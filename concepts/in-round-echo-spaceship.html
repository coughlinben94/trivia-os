In-round echo motif — spaceship flyby (midnight-galaxy)

Different pattern from the round-journey files in this folder — this one plays during a round, not between rounds. It's a small, rare, CSS-only ambient accent living in the persistent layer (ParticleBackground.jsx territory in the real app), not a theatrical full-stage moment. This mockup includes a fake trivia-question card so you can judge whether the flyby ever gets in the way of what a guest is actually reading.

What this is: per the "slight instances reminding guests where we're at" idea — a meteor streak, a nova pulse, a spaceship flyby — this is the third example of the same underlying mechanism: a small, theme-flavored accent that plays occasionally during a round, distinct from the big theatrical round-journey moments between rounds. Same mechanism, different content — a meteor-streak echo or a nova-pulse echo for other themes would be built the same way, not a new system per idea.

Why this is CSS-only, not canvas: round journeys get canvas/rAF freedom because they're one-shot and occasional. This plays inside a live round, on the same persistent layer that already has to run jank-free for a 3+ hour show — so it follows ambient-design-law.md's stricter rule: transform/opacity only, no continuous per-frame JS state. The flyby here is a single CSS @keyframes animation triggered by a class toggle; the only JS is a timer deciding when to trigger it, not driving the motion itself.

Why it stays out of the safe area: the dashed box marks where question text and answers actually sit. The flyby's path (top ~8-18% of frame, well above the card) is deliberate — a round-journey gets to use the whole stage because nothing needs to stay readable during it, but this plays while a guest is mid-read, so it has to stay atmosphere, never figure competing with the actual question. Worth watching a few flybys with a real question on screen to judge whether even this path is distracting enough to move higher or make smaller.

On cadence: the auto-mode toggle here fires every 4-9s so you can actually see it during a demo. In a real show that's much too often — a rare accent that shows up every few minutes reads as "oh, neat," the same one showing up every few seconds reads as noise. Real cadence needs its own decision (probably in the same "deliberate irregularity, not a fixed timer" spirit as the app's existing ambient timings), separately from this prototype.

No start/end point tied to a round: rounds vary in length — some have five questions, some fifteen — so anything scheduled as "once per round" or "at X% through the round" either fires inconsistently or requires knowing a round's length in advance, which this layer shouldn't need to know. The mechanism here is deliberately just a randomized timer running on its own clock, with zero awareness of round or slide boundaries — it doesn't start when a round starts or need to resolve before a round ends, the same "doesn't care what's structurally happening" spirit the persistent ambient layer already has for its own remount rule. The one edge case worth naming rather than solving here: a flyby could be mid-animation the instant a round-journey transition fires. Given a flyby is ~3s and a round is many minutes, this is rare and low-stakes — letting it just finish (or simply get visually covered by the journey's own full-stage takeover) is probably fine without special handling, but flagging it in case it reads as janky once this is live.

Open question for you: should in-round echoes be theme-specific content the same way journeys are (a meteor for midnight-galaxy, embers for autumn-harvest), or a single shared "something drifts by" mechanism reskinned per theme's palette? This prototype assumes the former, matching how journeys already work, but it's worth deciding explicitly rather than defaulting.
Self-gate
  • GPU-only: the flyby is transform + opacity via CSS @keyframes, no canvas, no continuous JS per-frame work — matches the ambient layer's stricter rule, unlike the round-journey files in this folder.
  • No pop-in: opacity ramps in/out at both ends of the flyby, it doesn't snap into view mid-frame.
  • Stays clear of the safe area: path runs well above the question-card region shown in this mockup.
  • Deliberately rare, not periodic: demo mode uses a randomized interval, not a fixed timer, per the loop-construction "deliberate irregularity" rule — though the real production cadence still needs its own decision, flagged above.
  • Real theme colors only: ship tint uses midnight-galaxy's highlight/accent/text values.
