Galaxy → Harvest — cross-theme handoff journey

The structural piece agreed on this session: a journey has two ends. Same theme both sides of a round boundary → play one continuous single-theme motif (that's the existing meteor-shower-to-galaxy file, unchanged). Different themes on either side → depart in the leaving theme's own motif, cross a short neutral bridge (theme-agnostic, so this isn't an N² combinations problem), then arrive in the entering theme's own motif. This prototype is that second case: midnight-galaxy (Round 1) handing off to autumn-harvest (Round 2).

Sequence (~5.75s): depart-hold (400ms, calm galaxy starfield, this is still "inside" Round 1) → meteors out (1300ms, the galaxy file's own departure motif, reused as-is, warp climbing) → bridge (750ms, a genuinely neutral beat — background tone crossfades from galaxy's deep purple-black to harvest's deep ember-black, a small set of drifting motes shift color from galaxy highlight to harvest's hot gold across their own lifetime so the handoff reads as one continuous idea rather than a hard cut, and a fine noise/mote layer stays over the near-black crossfade the whole time per the near-black-banding lesson) → converge (900ms, the harvest file's own gathering-embers motif, reused as-is) → supernova (1500ms, the harvest file's own flash/ring/burst, "Round 2" fades in around the 35% mark in harvest's own text color) → settle (900ms, calm harvest ambient, label holding).

What's reused vs. new: the galaxy half reuses the meteor-shower-to-galaxy file's star/meteor functions unmodified; the per-phase warp target values were re-tuned for this journey's own pacing (departing into a bridge reads differently than departing into a settle), so "reused" means the math, not every tuned number. Same relationship for the harvest half and its converge/burst functions. The only genuinely new code is the bridge phase (background tone crossfade + color-transitioning motes) and the single shared label element (one #roundLabel, not two elements — an earlier version of the meteor-shower file had a real bug where #round1 was never hidden before #round2 appeared, since fixed there; this file never had two label elements to begin with, so that class of bug structurally can't happen here).

Reduced motion: no meteors, no converge/burst, no motes, no windshield gag — the background tone crossfade only runs during the 750ms bridge window (not stretched across the whole ~5.75s), and the round label appears at the same relative point (35% into where the supernova beat would be) as the non-reduced version's label timing.

Everything from this session, combined: this file now carries every piece built tonight in one continuous journey — the meteor shower and windshield-crack-then-bandaid story beat from the standalone spaceship file (ported in as-is, just re-timed to this file's shorter 1300ms meteors-out window instead of that file's 1450ms), the cinematic nova upgrade (wispy filament ring, anamorphic core streak, extended hold-at-peak) on the harvest arrival, and the fix that keeps background warp out of the meteor beat so the shower stays a clean isolated moment. Nothing here is a new mechanic — it's the existing pieces assembled into the one thing they were always meant to add up to.

Open question for you: this prototype always spends a fixed 750ms in the neutral bridge regardless of which two themes are handing off. Some theme pairs might want a longer or differently-styled bridge (a theme with a very bright palette crossfading through near-black might read harsher than two dark themes meeting) — worth deciding whether bridge duration/style is a fixed constant or a per-pair-of-themes tunable once more handoffs exist.
Self-gate
  • Two-ended structure matches this session's agreed design: departure motif (leaving theme, native colors) → neutral bridge (theme-agnostic) → arrival motif (entering theme, native colors).
  • Both halves reuse already-built, already-gated motifs rather than inventing new per-theme math — the bridge is the only new mechanism, and it's intentionally theme-agnostic so it doesn't need to be rebuilt per theme pair.
  • Real theme colors only, both themes, straight from themes/index.js.
  • Replay button and reduced-motion toggle both tested working.
  • Single label element — the two-label overlap bug from the earlier file structurally can't recur here.
  • Near-black banding guarded: bridge keeps a mote/noise layer over the whole crossfade rather than a bare gradient ramp through near-black.
