# Folder Guide — Trivia OS

Plain-English map of this project, so you can find your way around without a coding background.

## What this is
The real-time platform that runs Baynes trivia nights — the host control surface, the `/display` TV renderer for the taproom screens, `/join` for players' phones, rounds, scoring, powerups, ambient animated backgrounds/themes, and slides (including the automated Winner Reveal). Built with React + Vite and Supabase Realtime.

## How to run it on your computer
1. Open a terminal in this folder.
2. `npm install` (first time only)
3. `npm run dev` — opens it in your browser (Vite prints the address).

## What's in each folder
- **client/** — the actual app the host, TVs, and phones see. This is the heart of the project (screens, slides, themes, scoring). Most changes happen here.
- **api/** — small server-side functions.
- **supabase/** — the database (shows, scores, questions) definitions and setup.
- **references/** — design and behavior notes the app is built against (e.g. slide specs).
- **trivia-questions/** — question content/material.
- **docs/** — written notes, plans, and the dress-rehearsal checklist.
- **scripts/** — developer helper tools (e.g. the theme audit).
- **e2e/** and **tests/** — automated checks that click through the app.
- **public/** — static files served directly to the browser.
- **dist/**, **test-results/**, **theme-screenshots-v2/** — auto-generated output. Safe to ignore; can be deleted and regenerated.

## Key files at the root
- **PRODUCT.md** — who uses it and how (the five viewer types).
- **AUDIT.md** / **theme-audit-report.md** — notes and generated reports about theming (the report is regenerated by `scripts/theme-audit.js`).
- **CLAUDE.md** / **SKILL.md** — instructions that help an AI assistant work on this project.
- **package.json** — project settings and `npm` commands.
- The `*.config.js` files — tooling settings; leave alone unless changing setup.

## A note on the two extra folders you may remember
`trivia-os-logo` and `trivia-os-shiny` were **not** separate projects — they were linked git "worktrees" of this same repo, each holding one branch of work. Those folders have been removed; the work lives safely as branches inside this project's git history (`fix/baynes-logo-asset` and `fix/shiny-gold-signal`). One tidy-up command remains — see `~/Projects/README.md`.
