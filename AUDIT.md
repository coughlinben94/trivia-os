# Trivia OS — Full System Audit

**Date:** 2026-07-05
**Method:** Direct code investigation (build, grep, targeted reads, `git log` verification) across the full `client/src` tree. Three parallel background subagents were dispatched to cover the theme deep-dive, wiring audit, and bug-hunt/UX sections in parallel; all three stalled twice on an infrastructure watchdog issue unrelated to the task content. Rather than keep retrying, the remaining ground was covered directly in the foreground. See **Scope Note** at the end for what that means for coverage.

## 1. Executive Summary

**Health grade: B+ — solid foundation, no crash-class bugs found, real but fixable live-show risk.**

Zero P0 (crash/show-breaking) findings turned up under direct code reading — that's the headline, and it's genuinely good news for a platform running live in front of a paying crowd. The core architecture (per-show theme overrides, Realtime sync, the font-upload lifecycle, the Final Break/Winner Reveal auto-close, error boundaries on `/display`, timezone handling) is well-built and, in several places, more defensively coded than the documentation gives it credit for.

The real risk in this codebase right now is **documentation drift**, not code defects: several "critical rules" and "known mechanisms" described in SKILL.md/references no longer match what's actually running, in ways that could cause a future session (human or agent) to redo already-shipped work, hunt for UI controls that don't exist, or trust a status marker that's wrong.

**Top 5 risks before the next live show:**

1. **WIRE-1 (P1)** — Opening the Scoreboard admin modal doesn't suppress the Stream Deck/keyboard shortcuts the way the old Score panel does. A stray key after clicking a non-input control inside the modal can advance the live slide or toggle the TV scoreboard overlay while the host is looking at the wrong screen.
2. **THEME-1 (P1)** — The TV scoreboard overlay (`ScoreboardOverlay.jsx`) hardcodes `Boogaloo` at all 3 of its text sites and completely ignores per-show font overrides — the one surface where a custom font pick visibly doesn't apply.
3. **WIRE-2 (P1, doc)** — The documented "Final Break" checkbox mechanism doesn't exist in the code at all. The real mechanism (fully automatic, no checkbox) works differently and is undocumented — a real risk of confusion or duplicated effort next time someone touches show-closing logic.
4. **DRIFT-1 (P1, doc)** — 4 of the 9 bespoke ambient themes have already shipped reworks (confirmed via `git log`) but are marked as still needing work in both SKILL.md and themes.md — risking a future session re-doing finished work.
5. **BUG-1 (P2)** — Winner Reveal silently ends with no name/points/confetti if zero teams ever recorded a score — an unlikely but real edge case with no error and no crowd-facing explanation.

Everything else is P2/P3: dead code, stale doc claims, and small cosmetic gaps. Build succeeds cleanly with only 3 minor warnings (dead-code duplicate keys in `Join.jsx`).

## 3. Theme System — Deep-Dive Summary

Priority area, audited in the most depth:

- **Theme object shape**: all 21 themes have an identical, complete key set — no theme is missing a documented field. The shape is actually *more* complete than documented (see THEME-3's dead `particles`/`cutawayFrame` fields).
- **Fallback theme IDs**: contrary to 3 different docs' repeated warnings, this is already unified through one constant (`DEFAULT_THEME_ID`) — not three independently-drifted values (DRIFT-3).

- **Hardcoded fonts**: 2 real violations found (THEME-1, THEME-2), both on the `/display` side where it matters most (they'd visibly break a per-show font override on live TV). One suspected violation (`SlideElements.jsx`) turned out to be intentional per-element styling, confirmed via its real font-picker UI.
- **Ambient status markers**: confirmed via `git log` that the bespoke/gradient split itself is accurate, but 4 of 9 bespoke themes have wrong status markers (DRIFT-1).
- **Not fully covered in this pass**: an exhaustive frame-by-frame animation-law audit of all 9 bespoke ambients (anchor/drifter/atmosphere quality, safe-area precision) and the full hardcoded-hex-color sweep across every slide component. A GPU-only/reduced-motion spot-check came back clean, but wasn't exhaustive across all 21 theme branches. See Scope Note.

## Scope Note

This audit is thorough at the code-correctness and wiring level — every finding above was verified by actually reading the relevant code, running the build, or checking `git log`, not inferred from the docs alone. Three areas got less depth than the original brief called for, and it's worth being direct about why:

Three parallel background subagents were dispatched early on to cover the theme system's remaining items (hardcoded-color sweep, `autoFitText` family-mismatch check, full 21-theme `ParticleBackground` animation-law audit), the wiring audit, and the bug-hunt/UI-UX review. All three stalled twice in a row on an identical infrastructure watchdog failure ("no progress for 600s") unrelated to their actual task content — a systemic issue with background execution at the time, not a problem with the investigation itself. Rather than burn more time on a third retry with the same likely outcome, the highest-value remaining ground (keyboard guards, error boundaries, Final Break wiring, Winner Reveal edge cases, timezone handling, nullable-data patterns, the scoreboard debounce path, several hardcoded-font sites) was covered directly in the foreground instead, which is reflected in the findings above.

What that leaves genuinely thin: the full hardcoded-hex-color sweep across every slide component, the `autoFitText` family-mismatch check at every call site, an exhaustive (not spot-checked) pass over all 21 ambient themes' keyframes for GPU-only/reduced-motion compliance, and the entire UI/UX review (Section 5). None of these showed any red flags in the spot-checking that did happen, but "spot-checked clean" isn't the same claim as "exhaustively verified" — worth a follow-up pass if you want that last layer of confidence, particularly on the color sweep since that's the kind of thing that's cheap to check and cheap to fix if something turns up.
